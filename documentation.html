Getting Started

Welcome to Ocrservice.net API v1.0 documentation! This page describes how to use Image Pre-Processing services that let you develop client applications without diving deep into Computer Vision.

Authentication
Services are members only and use this link to sign up first. You will need to provide valid credentials every time you make a request. For this version, you can use your username and password for authentication.

Request/Response Formats
URL format that contains version number and service name follows as:
http://ocrservice.net/api/[version]/[service_name]
Example:
http://ocrservice.net/api/1.0/text_detection
JSON is used to interact with the server, so set Http header which is Content-Type and use only POST as method.
POST /api/1.0/text_detection HTTP/1.1
Host: ocrservice.net
Content-Type: application/json
...
Request:
{
  "username": "your_username",
  "password": "your_password",
  "url": "your_image_url",
  "return_type": "text"
}
Response:
{
  "status": "success",
  "unique_id": "1234567890",
  "params":
    {
      "username": "your_username",
      "return_type": "text",
      "url": "your_image_url"
    },
  "result": Array
}
Failed response:
{
  "status": "failed",
  "unique_id": "1234567890",
  "message": "Invalid credentials provided",
  "params":
    {
      "username": "your_username",
      "return_type": "text",
      "url": "your_image_url"
    }
}
Form posts are not accepted, so build your JSON object. Also, only way to send your image is via URL. (Multipart uploads are not supported)
Response can include a result array or URL of output image after processing depending on your return_type.
It will never return anything like raw or Base64 encoded image data. Output images will be accessible only to with URL without any authentication and they are stored for at least 1 day. They are deleted permanently after the expiration date.

Rate Limiting
Ocrservice.net has the right to limit number of requests can users make in an hour or day for fair usage policy. Numbers are not explicitly published yet, but still exists to prevent abusive use of services.

Text Detection

http://ocrservice.net/api/1.0/text_detection
Name Value Required Description
username string yes Username of your account
password string yes Password of your account
url string yes URL of image to be processed
return_type string yes Return type of output. text, image values are accepted
detection_algorithm string yes Choose detection algorithm between Neumann-Matas and Stroke Width Transform. neumann_matas, swt values are accepted
Return result or output_url Choosing text returns points x y width height of rectangles detected by algorithm and image returns URL of output image
